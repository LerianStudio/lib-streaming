# Plan: Reorganize lib-streaming Into a Proper Go Library Layout

## Goal

Turn the current flattened root package into a clean Go library structure:

- Keep `github.com/LerianStudio/lib-streaming` as the single public entry point.
- Move implementation details behind compiler-enforced `internal/...` packages.
- Do not introduce `pkg/`; it would create extra public import paths without enforcing boundaries.
- Move test-only helpers out of the production root API.
- Preserve behavior first; change organization, then only make API cleanup that supports the new entry point.

The user clarified there are no downstream users, so pre-v1 breaking cleanup is acceptable. Still, the root import path should remain the ergonomic production API.

## Target Layout

```text
lib-streaming/
  doc.go
  emitter.go              # Emitter, New, NewProducer, Producer facade
  options.go              # public With* wrappers / aliases
  types.go                # public aliases for model/config/manifest types
  errors.go               # public error classes, sentinels, IsCallerError
  outbox.go               # public outbox aliases + WithOutboxTx
  manifest.go             # public manifest/handler wrappers

  streamingtest/
    mock.go               # MockEmitter and Assert*/WaitForEvent helpers

  internal/
    contract/             # event model, catalog, policies, public sentinels
    config/               # Config, LoadConfig, env parsing
    manifest/             # descriptor, manifest document, HTTP handler
    cloudevents/          # Kafka CloudEvents header codec
    emitter/              # noop emitter implementation
    producer/             # franz-go producer runtime and all Producer methods
```

## Public Entry Point Shape

The root `streaming` package remains the API consumers import.

Use root aliases for value types where aliases keep the API simple and do not leak implementation:

```go
type Event = contract.Event
type EmitRequest = contract.EmitRequest
type Catalog = contract.Catalog
type Config = config.Config
type PublisherDescriptor = manifest.PublisherDescriptor
type DeliveryPolicy = contract.DeliveryPolicy
type OutboxEnvelope = contract.OutboxEnvelope
```

Use a root wrapper for `Producer` instead of `type Producer = producer.Producer`:

```go
type Producer struct {
    inner *producer.Producer
}
```

Root `Producer` delegates:

- `Emit`
- `Close`
- `CloseContext`
- `Healthy`
- `Run`
- `RunContext`
- `Descriptor`
- `RegisterOutboxRelay`

This keeps godoc and user code centered on `streaming.Producer`, while the runtime implementation stays private under `internal/producer`.

Root constructors:

- `New(ctx, cfg, opts...) (Emitter, error)` returns `NewNoopEmitter()` when disabled/no brokers, otherwise wraps `internal/producer.NewProducer`.
- `NewProducer(ctx, cfg, opts...) (*Producer, error)` always constructs the real producer and returns the root wrapper.
- `NewNoopEmitter() Emitter` returns the internal noop emitter as the public interface.
- `NewEmitRequest(request EmitRequest) (EmitRequest, error)` should be added because `AGENTS.md` and `CHANGELOG.md` already document it as public, while the current code only has unexported `newEmitRequest`.

Root should not expose `MockEmitter`; test helpers move to `streamingtest`.

## Dependency Direction

Enforce one-way dependencies:

```text
streaming root
  -> internal/contract
  -> internal/config
  -> internal/manifest
  -> internal/emitter
  -> internal/producer

internal/producer
  -> internal/contract
  -> internal/config
  -> internal/manifest
  -> internal/cloudevents

internal/config
  -> internal/contract

internal/manifest
  -> internal/contract

internal/cloudevents
  -> internal/contract

streamingtest
  -> github.com/LerianStudio/lib-streaming
```

Hard rule: no `internal/...` package imports the root `streaming` package.

Run read-only/baseline verification before moving files:

- `go list ./...`

If baseline already fails, record failures and distinguish pre-existing failures from refactor regressions.

### 2. Extract `internal/contract`

Move pure contracts and validation primitives first so other packages can depend downward without cycles.

Move these files or split their contents:

- `streaming.go` -> `internal/contract/errors.go` plus root `errors.go`
- `event.go` -> `internal/contract/event.go`
- `emit_request.go` -> `internal/contract/emit_request.go`

Also extract shared validation from `publish.go` into `internal/contract/validation.go`:

Keep producer-only policy helpers such as `directAllowed`, `outboxAlways`, and `dlqAllowed` unexported in `internal/contract` only if `internal/producer` needs them; otherwise move equivalent decisions into producer to avoid exporting implementation decisions.

Add `contract.NewEmitRequest` and root `streaming.NewEmitRequest` with copy-on-construction semantics for public callers. Keep an internal validation path for producer if it needs to avoid extra payload copies on the hot path.

At this stage root can still call old producer code until the runtime move happens. The facade should compile after every slice.

## Risks and Mitigations

- Import cycles: enforce that internal packages never import root.
- Method ownership: move all `Producer` receiver files together; Go methods must stay in the same package as `Producer`.
- Test churn: move white-box tests beside internals instead of weakening encapsulation for tests.
- Behavior drift: perform mechanical package moves first, then public entry point cleanup.
- Godoc ugliness: use a root `Producer` wrapper, not an alias to `internal/producer.Producer`.
- Hot-path allocations: keep a producer-internal `EmitRequest` validation path if public `NewEmitRequest` copies payload for caller safety.

## Non-Goals

- Do not add a `pkg/` directory in this pass.
- Do not change Kafka topic naming, CloudEvents wire format, outbox row format, metrics labels, or circuit-breaker behavior.
- Do not introduce backward-compatibility shims for mock helpers; no downstream users exist and this is pre-v1.
